[Q] repo:command not found for mac

· 2020-10-06 · # 环境/配置

一、为什么会遇到这个问题？

最近在使用Mac尝试编译Android源码时遇到的错误
在执行repo指令下载源码时终端抛出"repo:command not found"的错误

二、解决了吗？

解决了，我的解决方案是在终端窗口输入以下指令

echo 'export PATH=$PATH:$HOME/bin' >> ~/.bash_profile
export PATH=$PATH:$HOME/bin

接着再执行repo命令就不会报错了

三、你是如何解决的？

网上大部分的做法是在终端输入

echo 'export PATH=$PATH:$HOME/bin' >> ~/.bashrc  
export PATH=$PATH:$HOME/bin

这两行代码在只会临时有效，重新打开新终端窗口依旧报 command not found
为什么会这样？接着往下看

 echo 'export PATH=$PATH:$HOME/bin' >> ~/.bashrc

第一行指令中>>符号是追加内容操作，我们去用户目录下找到.bashrc这个文件

1.左上角访达->前往->前往文件夹->键入~/

2.进入到用户目录后按下Command+Shift+. 显示隐藏文件

3.找到.bashrc文件，右键使用文本编辑打开

4.如图所示，.bashrc文件的确已经追加PATH路径，说明指令是的确正确执行了
那为什么不生效呢？
Google一番后发现因为OS X在login启动时不会加载.bashrc文件，它的读取顺序是(不同版本可能会有差异)

  • /etc/profile
  • ~/.bash_profile
  • ~/.bash_login
  • ~/.profile

5.既然已经知道问题所在了，那么解决起来就比较简单了，以下两种方案任选一种即可

  • 方案一
   # 打开~/.bash_profile文件，在文件最后一行添加
   if [ -f ~/.bashrc ]; then
       source ~/.bashrc
   fi
  • 方案二
# 将PATH路径追加到.bash_profile中，打开终端输入
echo 'export PATH=$PATH:$HOME/bin' >> ~/.bash_profile

搞定！

四、总结

网上大部分博客的教程只能保证在当前窗口有效
既然在窗口中是可以执行repo命令，证明repo脚本是没问题的(有时候下载的repo文件大小为0)
那么我们根据平台的不同，想办法把repo加入到全局变量中即可
